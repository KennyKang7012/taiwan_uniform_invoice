# 統一發票中獎號碼爬取與靜態網頁呈現系統 - 軟體開發需求規格書 (SRS)

## 1. 專案概述
本專案旨在開發一個自動化工具，爬取財政部稅務入口網之「統一發票中獎號碼」，並將最近兩年內的歷史中獎號碼彙整後，以美觀且互動性佳的靜態網頁呈現給使用者。

## 2. 功能需求

### 2.1 資料爬取 (Data Crawling)
*   **目標來源**：財政部稅務入口網 (https://www.etax.nat.gov.tw/etw-main/ETW183W1/)。
*   **時間範圍**：爬取最近兩年（目前為 113 年至 114 年）的所有已開獎期別。
*   **篩選規則**：
    *   僅擷取列表頁中「序號」為**偶數**的項目（即「一般中獎號碼」，如「114年 09 ~ 10 月」）。
    *   排除序號為奇數的「中獎清冊」項目。
*   **詳細內容**：需進入期別詳情頁，爬取包含特別獎、特獎、頭獎等所有中獎號碼。依照法規第 3 條，包含：
    1.  **特別獎**：八位數完全相同 (1000萬)。
    2.  **特獎**：八位數完全相同 (200萬)。
    3.  **頭獎**：八位數完全相同 (20萬)。
    4.  **二至六獎**：依頭獎末七至末三位數完全相同而定。

### 2.2 資料處理 (Data Processing)
*   **格式化**：將爬取到的資料結構化（如 JSON 格式），記錄年份、月份期別、獎項名稱及對應號碼。
*   **分類**：依據期別進行分類排序，確保最新期別顯示在最上方。

### 2.3 靜態網頁呈現 (Static Web Presentation)
*   **技術栈**：HTML, CSS (Vanilla), JavaScript。
*   **介面設計**：
    *   **現代化設計**：使用動態效果、漸層配色與優質字體（如 Inter 或 Roboto）。
    *   **期別導覽**：提供清晰的期別列表（如 114年 09 ~ 10 月），點擊可快速查看對應中獎號碼。
    *   **響應式佈局**：確保在手機與電腦端皆有良好視覺體驗。
*   **無後端依賴**：最終產出為純靜態網頁（index.html 及相關資源），可直接使用瀏覽器開啟。

### 2.4 自動化更新機制 (Automation)
*   **定時觸發**：系統需能自動在每單月 25 日（開獎日）及其後兩日自動執行爬取。
*   **自動發布**：爬取完成後，自動更新網頁內容並同步至 GitHub Pages 託管環境。
*   **通告補足**：即使非開獎日，亦提供手動觸發機制以應對臨時更新。

## 3. 非功能需求
*   **穩定性**：應具備基本的 Network Error 處理機制，避免因暫時性連線問題導致爬蟲崩潰。
*   **效能**：網頁載入應迅速，圖片（如有）需優化。
*   **程式碼規範**：變數命名清晰，關鍵邏輯需附上繁體中文註釋。

## 4. 技術規格
*   **程式語言**：Python 3.12+。
*   **環境管理**：使用 `uv` 管理虛擬環境與套件依賴。
*   **爬蟲套件**：`requests`, `beautifulsoup4` (或 `playwright` 視動態加載需求而定)。
*   **開發語言**：所有文件、註釋及對話全程使用**繁體中文**。

## 5. 驗證標準
*   爬取的號碼必須與官網完全一致。
*   網頁需能正確切換並顯示不同期別的資訊。
*   `uv run` 應能一鍵完成爬取並生成網頁。
